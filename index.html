What this is

One tool for the math you do after every experiment.

If you run product, marketing, or growth experiments, you already know the drill: an experiment finishes, and now someone needs to figure out whether the difference is real or just noise. This app does that — with the right test for your data type, real confidence intervals, and a plain-language verdict instead of a bare p-value.

📊

Two ways to test significance

Classic frequentist tests (z-test, Welch's t-test) alongside a Bayesian mode that gives you a direct probability instead of a p-value.

🧮

Plans your sample size too

Before you launch, find out how many users per group you actually need to detect the lift you care about.

📁

CSV in, verdict out

Type numbers in by hand for a quick check, or upload a CSV export straight from your database or analytics tool.

Two approaches, one answer

Frequentist or Bayesian — pick what your team speaks.

Both modes are statistically sound. They just phrase the conclusion differently — and the app supports both so you're not locked into one.

Frequentist mode

"p = 0.03"

Answers: how surprising would this data be if there were truly no difference?

  • Conversion: two-proportion z-test
  • Continuous: Welch's t-test (unequal variance safe)
  • Also reports: confidence interval, statistical power, Cohen's d
  • Good for: regulated industries, academic-style reporting, teams already fluent in p-values

Bayesian mode

"92% chance B wins"

Answers: how likely is it that the variant is actually better? — directly.

  • Conversion: Beta-Binomial conjugate model (exact, instant)
  • Continuous: Bayesian bootstrap (no normality assumption)
  • Also reports: credible interval, expected uplift, posterior charts
  • Good for: stakeholder-facing readouts, product/growth teams, intuitive go/no-go calls

How to use it

From repo to result in under five minutes.

Install the dependencies

Everything the app needs is pinned in requirements.txt — Streamlit for the interface, pandas/numpy/scipy for the math, Plotly for the charts.

pip install -r requirements.txt

Launch the app

This starts a local web server and opens the analyzer in your browser — nothing gets sent anywhere, it all runs on your machine.

streamlit run app.py

Choose your test type

In the sidebar: pick Frequentist or Bayesian, then choose Conversion Rate (binary: clicked / didn't), Continuous Metric (a number per user, like revenue), or Sample Size Calculator if you haven't launched the test yet.

Enter your data

Type in totals manually for a quick gut-check, or upload a CSV. For conversion tests, the CSV needs a group column (control/variant) and a converted column (0 or 1). For continuous metrics, swap converted for value.

group,converted control,0 control,1 variant,1

Read the verdict

Click Run Analysis. You'll get the headline numbers, a chart, and a plain-language recommendation — ship it, hold off, or collect more data — plus an expandable explanation of exactly what the statistics mean.

Questions

A few things worth knowing.

Does any of my data leave my computer?

No. Streamlit runs entirely on whatever machine you launch it on — local laptop, your own server, or your Streamlit Cloud account. There's no external API call in the statistics code itself.

Which mode should I trust — frequentist or Bayesian?

Both are mathematically valid; they just answer slightly different questions and report the answer differently. If your organization already reports p-values, stick with frequentist for consistency. If you want a number that's easier to explain in a meeting ("87% chance B wins"), use Bayesian. The app runs both off the same input data, so you can always sanity-check one against the other.

What if I don't have a CSV — can I just type in totals?

Yes. Both conversion and continuous modes support manual entry — just the visitor/conversion counts, or pasted-in numbers for continuous metrics.

Is this a replacement for a real statistician?

No — think of it as a fast, correct first pass. It won't catch issues like sample ratio mismatch, novelty effects, or seasonality on its own; that context still needs a human who knows the experiment.
